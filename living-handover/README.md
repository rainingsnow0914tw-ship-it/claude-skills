# 🐱 living-handover skill

**活的交接手冊** — 讓 Claude 在整個對話框過程中持續更新手冊，換框不再需要硬回憶。

---

## 這個技能解決什麼問題？

很多人只有在「準備換對話框」的最後一刻，才讓 AI 幫忙整理 handover。
這會出現幾個問題：

- 中途做了很多重要決策，但沒有即時記錄
- 到了要換框時，AI 已經忘了前面細節
- 手冊通常只剩大綱，缺少真正能接手的脈絡
- 新對話框常常還是要重新講一次

所以這個 skill 的核心思想不是：
**「最後生成一份交接」**，而是：

> **把交接手冊當成一份活文件，在整個工作過程中持續維護。**

---

## 它會做什麼？

啟用後，Claude 應該在對話推進中，持續幫使用者維護一份 handover，內容包括：

- 目前在做什麼
- 已完成哪些步驟
- 為什麼這樣決策
- 哪些檔案 / prompt / 規則已經確定
- 還剩哪些待辦
- 如果現在換框，下一個 Claude 要怎麼無痛接手

---

## 適用情境

這個 skill 很適合：

- 長流程專案
- 多輪 debug
- 多 AI 接力協作
- 需要隨時可能換框的任務
- 需要為未來自己保留脈絡的工作

---

## 成功標準

理想狀態下，這份 handover 應該做到：

- 不是只在最後一刻才補
- 即使中途打斷，也能快速恢復
- 新 Claude / 新對話框接手時不必從頭問
- 使用者看得懂目前進度，不會失去方向感

---

## 搭配建議

它很適合和：

- `code-documentation` 搭配使用 → 一個維護 code 內知識，一個維護全局狀態
- PRD / issue / checklist 搭配 → 形成完整 project memory
